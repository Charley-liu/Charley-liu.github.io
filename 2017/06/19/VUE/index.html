初解vue

日志二级目录
  • vue

摘要：关于vue的开发模式以及初步了解vue

正文：
　　关于vueJs，因为接触过ng1以及React，这两个前端框架我之前这两个框架做过小的应用，大致了解过用法以及流程，对于初学者而言的确难度挺大的，因为思维方式的不同，这两款框架都是由国外工程师构建的，所以理解上会有点难度，但是vue就不一样了，它的作者是一个中国人，尤雨溪（上海复旦附中）。。。膜拜一下，关于作者自行百度。
　　还有就是使用的MVVM模式开发的，不是ng1这样的MVC模式，

MVVM模式和MVC模式一样，主要目的是分离视图（View）和模型（Model），有几大优点

  1. 低耦合。视图（View）可以独立于Model变化和修改，一个ViewModel可以绑定到不同的”View”上，当View变化的时候Model可以不变，当Model变化的时候View也可以不变。
  2. 可重用性。你可以把一些视图逻辑放在一个ViewModel里面，让很多view重用这段视图逻辑。
  3. 独立开发。开发人员可以专注于业务逻辑和数据的开发（ViewModel），设计人员可以专注于页面设计，使用Expression Blend可以很容易设计界面并生成xml代码。
  4. 可测试。界面素来是比较难于测试的，而现在测试可以针对ViewModel来写。

　　边的这段话是引自于百度的，如果想多了解的话，可以看一下，关于vue的语法，基础很重要，相对于ng和react是比较容易上手的，地址在这vue官网，可以大致了解一下。今天就先了解一下vue，后续更新项目的搭建和配置。。。
